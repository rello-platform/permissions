# @rello-platform/permissions

Canonical permission registry for the Rello ecosystem. A single source of truth for the inter-app permission slugs that gate every service-to-service auth boundary across Rello + 10 spokes + 6 engines. Consuming this package makes a permission rename a **compile error** in every consumer instead of silent 401/403 drift.

## Install

```bash
npm install "github:rello-platform/permissions#v0.1.0"
```

## Usage

```ts
import {
  PERMISSIONS,
  ALL_PERMISSION_SLUGS,
  SLUG_TO_KEY,
  isPermissionSlug,
  type PermissionDefinition,
  type PermissionKey,
  type PermissionSlug,
} from "@rello-platform/permissions";

// Receiver call site:
const auth = await requireServiceBearer(request, {
  permission: PERMISSIONS.ENGAGEMENT_READ.slug,
});

// Picker UI:
for (const [key, def] of Object.entries(PERMISSIONS)) {
  console.log(def.label, def.description, def.validatedBy, def.grantedTo);
}

// Type guard for runtime validation at boundaries:
if (!isPermissionSlug(input)) {
  throw new Error(`Unknown permission slug: ${input}`);
}
```

## Shape

```ts
interface PermissionDefinition {
  readonly slug: string;          // wire format, e.g. "engine:access"
  readonly label: string;         // short human label, e.g. "Engine access"
  readonly description: string;   // one-sentence operational description
  readonly validatedBy: readonly PlatformSlug[];   // CI-derived: which receivers gate on this
  readonly grantedTo: readonly PlatformSlug[];     // hand-authored: which callers' keys carry this
}
```

`validatedBy` is generated by `scripts/regenerate-validatedby.ts` against pinned consumers (see `scripts/pinned-consumers.json`). Authors leave it empty in PRs; CI populates on each version bump. Drift between receivers and the registry is structurally impossible.

`grantedTo` is hand-authored from current `ApiKey` table state and refined as picker UI surfaces it.

## Migrations

`migrations/` contains generated SQL for the Postgres CHECK constraint that enforces canonical membership on `ApiKey.permissions` at the DB layer. v0.1.0 ships the draft; consumers ingest the migration when they bump to a version that applies it (see the canonicalization spec for the phase that flips this on).
